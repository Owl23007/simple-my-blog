# 第三章 UML 基础与建模实践

## 3.1 UML 简介与应用场景

UML（Unified Modeling Language）是一种标准的建模语言，用于描述软件系统的结构和行为。

**应用场景：**
- 系统设计与分析
- 团队沟通
- 文档记录

## 3.2 类图（Class Diagram）：结构建模核心

类图用于表示系统中的类及其之间的关系：

- **继承关系**（Inheritance）
- **关联关系**（Association）
- **聚合关系**（Aggregation）
- **组合关系**（Composition）
- **依赖关系**（Dependency）
- **实现关系**（Realization）

## 3.3 时序图（Sequence Diagram）：行为交互分析

时序图用于展示对象之间的交互顺序和时间。

## 3.4 状态图（State Diagram）：状态机建模

状态图用于描述对象在不同状态间的转换。

## 3.5 组件图与部署图简介

- **组件图**：描述系统的物理结构
- **部署图**：展示系统的部署架构

## 3.6 如何用 UML 表达设计模式？

设计模式可以用UML类图清晰地表达。
