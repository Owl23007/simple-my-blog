# 第五章 结构型设计模式

结构型设计模式关注对象和类的组合，形成更大的结构。

## 5.1 适配器模式（Adapter）

将一个类的接口转换成客户端所期望的另一个接口。

**适用场景：**
- 集成第三方库
- 接口兼容性问题

## 5.2 桥接模式（Bridge）

将抽象与实现分离，使它们可以独立地变化。

**优点：**
- 提高系统的可扩展性
- 避免过深的继承层次

## 5.3 组合模式（Composite）

将对象组合成树形结构以表示"部分-整体"的层次关系。

**适用场景：**
- 树形菜单结构
- 文件系统

## 5.4 装饰模式（Decorator）

动态地给对象添加新的职责，而不改变其原有的代码。

**优点：**
- 灵活地添加功能
- 比继承更加灵活
